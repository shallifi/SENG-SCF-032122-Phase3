    Introduction

    Welcome to the Dog Walker API! You can use our API to access Dog Walker API endpoints, which can get information on various dogs, their walks and feedings stored in our database.

    We have language bindings in JavaScript (fetch) and CURL! You can view code examples in the dark area to the right, and you can switch the programming language of the examples with the tabs in the top right.

    Dogs

    Get All Dogs

    fetch('http://localhost:9292/dogs')
  .then(response => response.json())
  .then(console.log)

    curl "http://localhost:9292/dogs"

    The above command returns JSON structured like this:

    [
  {
    "id": 1,
    "name": "Lennon Snow",
    "birthdate": "2020-08-31",
    "breed": "Pomeranian",
    "image_url": "https://res.cloudinary.com/dnocv6uwb/image/upload/v1609370267/dakota-and-lennon-square-compressed_hoenfo.jpg",
    "age": "2 years"
  }
]

    This endpoint retrieves all dogs.

    HTTP Request

    GET http://localhost:9292/dogs

    Query Parameters

    Parameter Default Description
    include_dog_walks false If set to true, the result will also include this dog's dog_walks. Each will then look like the JSON displayed in the documentation for the /dogs/:id endpoint
    breed nil If the breed parameter is included, the result will include only dogs matching the provided breed. Otherwise, all dogs will be included.

    This endpoint creates a new dog and responds with it.

    HTTP Request

    POST http://localhost:9292/dogs

    Request Body Parameters

    Parameter Description
    name (string) The new dog's name
    birthdate (string) The new dog's birthdate in "YYYY-MM-DD" format
    breed (string) The new dog's breed
    image_url (string) an image_url for the new dog

    This endpoint updates a specific dog in the database and responds with the updated version of the dog's information.

    HTTP Request

    PATCH /dogs/<ID>

    URL Parameters

    Parameter Description
    ID The ID of the dog to update

    Request Body Parameters

    All parameters below are optional, only included parameters will update the saved values for this dog.

    Parameter Description
    name (string) The new name of the dog
    birthdate (string in "YYYY-MM-DD" format) The new birthdate of the dog
    breed (string) The new breed of the dog
    image_url (string) The new image_url of the dog

    This endpoint retrieves all walks stored in our database. It also accepts a boolean parameter for recent which will configure the endpoint to respond with only walks in the last 4 hours if added.

    HTTP Request

    GET http://localhost:9292/walks

    URL Parameters

    Parameter Default Description
    recent (boolean) false If true, responds with all walks within the last 4 hours, if false, responds with all walks

    This endpoint creates a new walk, associates existing dogs with it by their ids and responds with information about the walk and all associated dogs.

    HTTP Request

    POST /walks

    Request Body Parameters

    Parameter Description
    time (string in "YYYY/MM/DDTHH:MM" format) The time of the newly created walk
    dog_ids (array of integers) an array of dog ids for the dogs that are going on this new walk

    Delete a specific Walk

    fetch('http://localhost:9292/walks/1',{
  method: 'DELETE'
})
  .then(response => response.json())
  .then(console.log)

    curl -X DELETE http://localhost:9292/walks/8

    The above command returns JSON structured like this:

    {
  "time": "2022-05-31T11:52:00.000Z",
  "dog_ids": [
    1,
    2
  ]
}

    This request removes a Walk from the database and returns the walk formatted in such a way that it can be used to undo the deletion and create the walk again (if the body of this response is sent as the body of a POST request to /walks).

    HTTP Request

    DELETE /walks/<ID>

    URL Parameters

    Parameter Description
    ID The ID of the walk to delete

    This endpoint creates a new DogWalk. It is used when interacting with a particular dog and logging a walk for that dog.

    HTTP Request

    POST /dog_walks

    Request Body Parameters

    Parameter Description
    dog_id (integer) The ID of the dog to be walked
    walk_time (string in "YYYY-MM-DDTHH:MM" format) The time of the walk to be created
    pooped (boolean) A boolean value indicating whether or not the Dog pooped on this walk

    Update a specific DogWalk

    fetch("http://localhost:9292/dog_walks/1", {
  method: "PATCH",
  headers: {
    "Content-Type": "application/json"
  },

    pooped: true,
    walk_time: "2022-05-31T11:30"
  })
})
  .then(response => response.json())

    curl -X PATCH -H "Content-Type: application/json" \
 -d '{
    "pooped": true,

 http://localhost:9292/dog_walks/1

    This endpoint updates a specific dog_walk in the database and responds with the updated version of the dog_walk's information.

    HTTP Request

    PATCH /dog_walks/<ID>

    URL Parameters

    Parameter Description
    ID The ID of the dog_walk to update

    Request Body Parameters

    All parameters below are optional, only included parameters will update the saved values for this dog_walk.

    Parameter Description
    pooped (boolean) whether or not the dog pooped on this walk
    walk_time (string in "YYYY-MM-DDTHH:MM" format) the time that this dog went on this walk

    Delete a specific DogWalk

    fetch('http://localhost:9292/dog_walks/17',{
  method: 'DELETE'
})
  .then(response => response.json())
  .then(console.log)

    curl -X DELETE http://localhost:9292/dog_walks/17

    The above command returns JSON structured like this:

    {
  "id": 17,
  "pooped": false,
  "dog_id": 1,
  "walk_id": 11
}

    This request removes a DogWalk from the database and returns the DogWalk formatted in such a way that it can be used to undo the deletion and create the DogWalk again (if the body of this response is sent as the body of a POST request to /dog_walks).

    HTTP Request

    DELETE /dog_walks/<ID>

    URL Parameters

    Parameter Description
    ID The ID of the dog_walk to delete

    Errors

    These errors may occur as you work with this API, if you see a 500 Internal Server error while you work, check the server logs in the terminal that is running your development server for some ideas about what went wrong.

    The Dog Walker API uses the following error codes:

    Error Code Meaning
    404 Not Found -- The specified record could not be found.
    500 Internal Server Error -- We had a problem with our server. Try again later.
